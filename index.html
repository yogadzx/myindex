Git 简单归纳

git 初始化
新建项目后，使用 terminal 在项目根目录下:
$ git init
进行 git 代码仓库初始化，此目录下会新建 .git 目录,此时本地 git 初始化完成。
git 常用命令
git add（ 将修改文件添加到暂存区 ）
git commit（ 提交本次修改，将暂存区合并到分支，-m"do something" 引号里面输入本次提交所更改到内容描述 ）
git checkout + 分支名称（ 切换到输入名称分支，git checkout -b 名称= git branch 名称 + git checkout 名称 ）
git checkout + 文件名（ 将该文件未add 的内容还原到上一次commit后 ）
git branch 查看项目本地分支（ -a 查看本地和远程分支 ）
git branch +名称（ 创建名称为...的新分支 ）
git log （ 查看每次commit的信息 ）
git reset --hard commit id （ 每次提交的id使用git log查看 ）
git status （ 查看工作区暂存区状态 ）
git push （ 将提交合并至远程仓库 ）
git pull （ 从远程仓库更新提交内容 ）
git 缩写命令
每个项目初始化后，会在项目的根目录下创建 .git 目录，进入目录使用文档编辑器打开 config 文件，如果没有 [alias] ，在最后一行新加 [alias] ，输入你想要缩写的单词例如：
[core] ... [alias] co = checkout
ci = commit
br = branch
st = status
...
保存，注意 ⚠️本次修改只限于当前项目。每个项目都有自己的 .git 目录。
git 创建远程仓库
以 github 作为远程仓库为例
首先确定本机是否有 ssh key (安全外壳协议),在Users／（ 用户名 ）／下是否存在 .ssh 目录？
如果存在证明本机存在ssh key，如果没有该目录说明本机还未创建ssh，使用:
$ ssh-keygen -t rsa -C "youremail@example.com"
经过三次回车后，进入User／（ 用户名 ）／.ssh 目录，一个名称为 id_rsa 的文件是私有秘钥，一个名称为 id_rsa.pub 的文件是公共秘钥，添加 sshkey 时请使用公共秘钥，私有秘钥泄漏会增加您信息的风险。
打开 github 登陆你的github后，点击头像在列表中选中 settings 。
在左侧列表选中 SSH and GPG keys，找到SSH keys 区域，点击 New SSH key 按钮，弹出如下输入框:
Title ：[ 任意输入 （ 能够描述当前用户 ）]
Key ：[ 将 id_rsa.pub 文件全部复制到这里。]
点击 Add SSH key ，完成！本台计算机有权访问 github 仓库。
现在创建一个 github 仓库。
点击 github 头像旁的 + ，在弹出列表中选择 New repository ，输入 Repository name
有两种方法进行本地与远程同步：

1.本地项目在 github 上新建仓库
不需要勾选 Initialize this repository with a README 。
2.github 新建项目，克隆到本地
勾选 Initialize this repository with a README 。

选择适合的方法后，点击 Create repository （ 默认为公共仓库，其他人可见，私有仓库请选择 Private 并支付money ）。根据所选择的方法不同，点击按钮后分为对应的两种步骤。
1.本地项目在 github 上新建仓库，Terminal 本地项目位置，输入例如：
git remote add origin https://github.com/github用户名/Repositoryname仓库名称.git
git push -u origin master
2.github 新建项目，克隆到本地，Terminal 项目目标位置，输入github项目中绿色按钮" Clone or download "的内容，例如：
git clone https://github.com/github用户名/Repositoryname仓库名称.git
